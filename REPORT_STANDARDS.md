# 群聊日报生成标准

## 强制执行标准

### 1. 数据完整性
- **必须分析所有聊天记录文件** - 一个文件都不能漏
- 统计并记录：总消息数、唯一发言用户数

### 2. 话题数量要求
- **每个核心话题必须包含 ≥10 条相关聊天记录**
- 聊天记录必须真实来自该话题的讨论

### 3. 描述内容标准
- 核心话题描述字数 ≥80 字
- 描述内容必须包含：背景信息、讨论细节、群友观点
- 不能简单敷衍，必须充实

### 4. 输出格式
- HTML 报告格式统一
- 包含活跃度概览、核心话题、深度洞察、资源分享
- GitHub Pages 部署

## 报告检查清单

生成报告前必须确认：
- [ ] 所有消息文件已统计
- [ ] 总消息数和用户数已更新
- [ ] 每个话题 ≥10 条聊天记录
- [ ] 话题描述字数达标
- [ ] 布局美观、无CSS错误
